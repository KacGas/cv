Test-Solving Page

Description: This project is a web application that allows users to take a timed test, answer questions, navigate between questions, and submit the test upon completion. It stores results and statistics in local storage, providing a user-friendly interface for managing tests.

Technologies Used:

  • HTML: Used for the basic structure and layout of the page.
  • TypeScript: For handling logic such as user interaction, test management, and timing.
  • JSON: Used for loading questions and answers into the test dynamically.

Key Features:

  • Test Navigation: Users can navigate between questions using "Next" and "Previous" buttons.
  • Timer: A timer starts when the user clicks "Start" and tracks the time spent on each question.
  • Mandatory Completion: The test cannot be submitted until all questions are answered.
  • Question Mode: Once an answer is selected and the user navigates away from the question, it is locked in reading mode.
  • Results & Statistics: Statistics, including time spent per question, are collected and stored in the browser's localStorage.
  • JSON-Based Questions: Questions and answers are loaded dynamically from a JSON file.

Administration Panel for Movies

Description: This project is a web application for managing movies with role-based access control. Users can view movies, while administrators can add, edit, and delete movies, as well as manage users through secure authentication and role management.

Technologies Used:

  • Java 17: Used for the backend logic, providing modern Java features for better performance and scalability.
  • Spring Boot 3.4: Provides a robust backend framework for building secure and scalable applications.
  • JWT Authentication: JSON Web Tokens are used for secure user authentication and session management.
  • MySQL: Database for storing user and movie data, ensuring smooth data management and retrieval.
  • Angular: Frontend framework for creating a dynamic and responsive user interface for managing movies and users.

Key Features:

  • Secure Authentication: Users authenticate using JWT-based security, ensuring secure login and access.
  • Role-Based Access Control: Admins can manage users, movies, and roles, while regular users only have viewing access.
  • Movie Management: Admins can add, edit, or delete movies from the database.
  • User Management: Admins can create and manage users and their roles (admin/user).
  • REST API: The system uses a REST API to provide secure and controlled access to the movie data and user management functionalities.
